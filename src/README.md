# /src - FPGA-BoardlessBench (F-BB) 実装層（コアコンポーネント）

このディレクトリには、シミュレーション・インフラのソースコードが収められています。
**「開発者が直接編集するファイル」**と**「DTSから自動生成されるファイル」**が混在しているため、以下の分類に注意してください。
※自動生成されるファイルは `make clean` によって削除されます。直接編集してもビルド時に上書きされるため、変更が必要な場合は必ずソース（DTS等）を修正してください。

## ディレクトリ構成と役割

| ディレクトリ | 役割 | 主なファイル | 生成種別 |
| :--- | :--- | :--- | :---: |
| **controller** | バックエンド管理 | `vlogic_controller.py` | 手書き |
| **shim** | システムコール横取り | `libfpgashim.c` | **自動生成** |
| **sim** | Verilator実行環境 | `sim_main.cpp` | **自動生成** |
| **rtl** | 回路ロジック (Verilog) | `vfpga_top_skeleton.v` | **自動生成** |
| | | `vfpga_top.v` | 手書き/コピー |
| **include** | 共有ヘッダー | `vfpga_config.h` | **自動生成** |

## 各コンポーネントの詳細

### 1. controller (Python)
- **vlogic_controller.py**: プロジェクトの心臓部。共有メモリの初期化、RTLシミュレータの起動監視、UARTブリッジの管理を行います。

### 2. shim (C)
- **libfpgashim.c**: アプリケーションの `open`, `mmap`, `ioctl` をフックして共有メモリにリダイレクトします。
- **注意**: このファイルはビルド時に `gen_vfpga.py` によって上書きされるため、直接編集しないでください。

### 3. sim (C++)
- **sim_main.cpp**: Verilator でコンパイルされた回路を 1 クロックずつ進めるためのメインループです。共有メモリのレジスタ値を読み取り、RTLの信号線へ入力します。

### 4. rtl (Verilog)
- **vfpga_top_skeleton.v**: DTSの定義に基づいた AXI Lite 風のバスインターフェースを備えたスケルトンです。
- **vfpga_top.v**: 実際にシミュレーションで使われるトップモジュールです。
  - **重要**: このファイルは実行スクリプト（`scenario_runner.sh`）によって自動的に配置されます。
    - シナリオ内に `vfpga_top.v` がある場合：そのファイルがここにコピーされます。
    - シナリオ内に `.v` がない場合（I2CやUART等のPS側機能のテスト）：自動生成された `vfpga_top_skeleton.v` がここにコピーされることで、最低限のバス接続が維持されます。
  - そのため、ここを直接編集しても実行時に上書きされます。変更が必要な場合は、各シナリオ内のファイル、あるいは大元のDTSを修正してください。

### 5. include (C)
- **vfpga_config.h**: ソフトウェア(C)とハードウェア(Verilog)の両方で共通して使われるアドレス定義などが含まれます。
