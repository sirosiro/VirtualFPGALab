# Project: VirtualFPGALab Specification

## 1. プロジェクトの核心的意図 (Core Intent)
本プロジェクトの目的は、Xilinx Zynq等のFPGA SoC開発において、物理的なボードへの依存を最小化し、ソフトウェア・エンジニアがWSL2環境でFPGAロジックを含めた開発・デバッグを完結させることにある。
システムコール・フッキング技術を用いてハードウェアの振る舞いを抽象化し、「ハードウェア完成を待たずにソフトウェアのロジック検証を回す」ことを究極の意図とする。

---

## 2. ライセンス戦略と設計モード (License Strategy)
- **Mode:** Hybrid-Bridge Mode (デフォルト適用)
- **方針:** 既存の標準的なLinuxカーネルインターフェース（uio, i2c-dev）との互換性を保ちつつ、エミュレーション・ロジック自体は完全に独立したクリーンな実装とする。

---

## 3. システム・コンポーネント設計 (Component Design Specifications)

### 3.1. Syscall Interceptor Layer (Shim)
- **責務:** アプリケーションが発行するデバイスアクセス関連のシステムコールをインターセプトし、仮想的なデバイスノードへリダイレクトする。
- **設計理由:** アプリケーション側のバイナリ互換性を維持し、実機環境とシミュレーション環境の切り替えを容易にするため。
- **主要な手法:** [LD_PRELOAD](LD_PRELOAD.md) による動的ライブラリ・フッキング。

### 3.2. Shared Memory Register Emulator (Virtual Logic Space)
- **責務:** FPGA上のレジスタ空間を共有メモリ上に再現し、プロセッサ（PS）と回路ロジック（PL）間の通信をシミュレートする。
- **詳細仕様:**
    - **物理実体:** /dev/shm（共有メモリ）。
    - **動作:** アプリによる特定アドレスへの書き込みを検知し、シミュレータ側の論理を更新する。

### 3.3. Logic Visualization & Diagnostic Tool (The Dashboard)
- **責務:** 仮想デバイスへのアクセスログ、レジスタの状態遷移、およびバス上のパケットを可視化・解析する。
- **機能:**
    - リアルタイムでのレジスタ監視。
    - 異常系（エラーレスポンス、タイムアウト）のシミュレーション注入。

### 3.4. RTL-C++ Integrated Bridge ([Verilator](verilator.md) Interface)
- **責務:** HDLで記述された回路設計を[Verilator](verilator.md)等を用いてC++モデル化し、ソフトウェアシミュレーションにサイクル精度の論理を提供する。

---

## 4. 成立条件と不変条件 (Invariants & Pre-conditions)
- **透過性:** アプリケーション層からは、通信相手が実デバイスかシミュレータかを意識させないこと。
- **完全性:** 共有メモリ上のレジスタ操作はすべてログに記録され、デバッグの再現性が担保されていること。

---

## 5. 既知の未解決課題 (Known Open Issues)
- **割り込み通知の遅延:** WSL2上のユーザー空間プロセス間通信による、物理割り込みとの時間的乖離の許容範囲。
- **マルチプロセス競合:** 複数のシミュレーション対象アプリが同時に同一의 仮想レジスタ空間にアクセスする場合の排他制御。
