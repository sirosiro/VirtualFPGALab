# 1. FPGAの本質

FPGAは
**Field Programmable Gate Array** の略で、

* Field = 出荷後（現場で）
* Programmable = 書き換え可能
* Gate Array = 論理回路の集合

という意味です。

通常のCPUやマイコンは「決まった回路の上でソフトウェアを動かす」構造ですが、
FPGAは逆で、

> **回路そのものをソフトウェアで定義する**

というのが本質です。

---

# 2. Xilinxの位置づけ

Xilinxは長年FPGA業界のトップ企業の一つで、現在は
AMDに買収され、**AMDのFPGA部門**になっています。

代表的な製品ファミリ：

* **Spartan**（低価格・小規模）
* **Artix / Kintex**（中規模）
* **Virtex**（ハイエンド）
* **Zynq**（FPGA + ARM CPUのSoC）

---

# 3. 内部構造（ざっくり）

FPGAは以下のようなブロックで構成されています：

* **LUT（Look-Up Table）**
  → 任意の論理回路を実現
* **フリップフロップ**
  → 状態保持（レジスタ）
* **配線ネットワーク**
  → ブロック間の接続を動的に変更
* **DSPブロック**
  → 乗算・加算など高速演算
* **Block RAM**
  → 内蔵メモリ

つまり

> 「論理IC + 配線」を丸ごとプログラム可能

---

# 4. CPUとの違い（重要）

| 項目    | CPU / MCU | FPGA      |
| ----- | --------- | --------- |
| 実行単位  | 命令（逐次処理）  | 回路（並列処理）  |
| 並列性   | 低い        | 非常に高い     |
| 柔軟性   | ソフトで変更    | 回路ごと変更    |
| 開発難易度 | 比較的低い     | 高い（HDL必要） |

特に重要なのは：

> FPGAは「並列ハードウェア」なので、処理を同時に大量実行できる

### ② 通信

## 1. Verilogとは何か（定義）

Verilogはプログラミング言語ではなく：

> **回路（ハードウェアの構造と振る舞い）を記述する言語（HDL）**

です。

C言語との最大の違いはここ：

| 観点 | C言語      | Verilog |
| -- | -------- | ------- |
| 本質 | 手順を書く    | 回路を書く   |
| 実行 | CPUが逐次実行 | 全部同時に動く |
| 時間 | あまり意識しない | クロックが支配 |

---

## 2. 一番重要な誤解の排除

C言語的な感覚：

```c
a = b + c;
d = a + 1;
```

→ 上から順に実行

---

Verilogの基本：

```verilog
assign a = b + c;
assign d = a + 1;
```

→ **同時に存在する回路**

* 加算器1：b + c
* 加算器2：a + 1

つまり：

> 「命令」ではなく「配線と部品」

---

## 3. クロックの意味（ここが核心）

Verilogでは2種類の世界がある：

### ■ (A) 組み合わせ回路（Combinational）

```verilog
assign y = a & b;
```

* 入力が変わると即変化
* クロック不要

---

### ■ (B) 順序回路（Sequential）

```verilog
always @(posedge clk) begin
    q <= d;
end
```

* クロックで状態更新
* レジスタ（フリップフロップ）

---

### ■ イメージ

```id="csm0hx"
clk:  ↑   ↑   ↑
      |   |   |
      d   d   d
      ↓   ↓   ↓
      q   q   q
```

---

## 4. 最小例：カウンタ

```verilog
module counter (
    input clk,
    input rst,
    output reg [7:0] cnt
);

always @(posedge clk) begin
    if (rst)
        cnt <= 0;
    else
        cnt <= cnt + 1;
end

endmodule
```

---

### ■ これが意味する回路

* 8bitレジスタ（cnt）
* 加算器（+1）
* クロック同期更新

つまり：

> **「1クロックごとに+1する回路」**

---

## 5. C言語との対応付け（重要）

C的に考えると：

| 観点   | C      | Verilog |

| 実行主体 | CPU    | 回路      |

## 2. FPGAの正しいイメージ

あなたの言葉：

> 電気的に連続しているのでシーケンシャルに状態変化

これは半分正しく、半分違う。

---

### ■ 正しい部分

* 信号は物理的に連続している
* 伝播遅延がある
* 完全同時ではない

---

### ■ 重要な違い

FPGAの設計では：

> **“いつ状態を確定させるか”をクロックで定義する**

---

### 3. 本質モデル（これを掴むと一気に理解が進む）

FPGAはこう動く：

```id="1r6f5l"
(1) 前のクロックの状態がある
(2) 組み合わせ回路が全部同時に計算する
(3) 次のクロックで一斉に状態確定
```

---

### ■ 図で表現

```id="6m4g9q"
      ┌───────────────┐
      │ 組み合わせ回路 │ ← 全部同時に動く
      └──────┬────────┘
             ↓
        [レジスタ群]
             ↑
         クロックで更新
```

## 4. シーケンシャルに見える理由

あなたの違和感の正体はここ：

> 「カウンタとかは順番に進んでるじゃないか」

これは：

* 回路が順番に動いているのではなく
* **状態が離散的に更新されているだけ**

---

### ■ カウンタの本質

```verilog
cnt <= cnt + 1;
```

これは：

```id="u4hzdi"
毎クロック：
  cnt(次) = cnt(今) + 1
```

---

### ■ 重要な理解

* 加算回路は常に存在している
* 計算は常にされている
* **採用される値だけがクロックで決まる**

## 5. PLCとの対応（かなり良い対応）

| PLC    | FPGA    |
| ------ | ------- |
| スキャン周期 | クロック    |
| 接点条件   | 組み合わせ回路 |
| コイル更新  | レジスタ更新  |

かなり近い。

ただし違い：

* PLC：周期が遅い（ms）
* FPGA：クロックが高速（ns）

# 1. レジスタ設計（最小構成）

## ■ アドレスマップ

```text

        bit0 = start (W)

        bit2 = done  (R/W1C)

0x08 : OUTPUT
```

---

## ■ ポイント

* start：書いた瞬間に1クロックだけ使う（パルス化）
* busy：処理中フラグ
* done：完了通知（CPUがクリア）

# 2. Verilog（コアロジック）

# 3. 状態機械（FSM）

```verilog
localparam IDLE = 0;
localparam RUN  = 1;

reg state;

always @(posedge clk) begin
    if (!rstn) begin
        state <= IDLE;
        busy  <= 0;
        done  <= 0;
    end else begin
        case (state)
            IDLE: begin
                done <= 0;
                if (start_pulse) begin
                    state <= RUN;
                    busy  <= 1;
                end
            end

            RUN: begin
                // 仮の処理（数クロックで完了）
                output_reg <= input_reg + 1;

                state <= IDLE;
                busy  <= 0;
                done  <= 1;
            end
        endcase
    end
end
```
